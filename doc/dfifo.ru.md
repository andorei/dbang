# dfifo. Трансформация данных из файлов в файлы

	версия 0.4.0

Утилита `dfifo` для чтения данных из файлов предоставляет те же возможности, что `dput`, а для записи данных в файлы те же возможности, что и `dget`.

На вход поступают файлы CSV, XLSX, JSON и произвольных текстовых форматов, на выходе создаются файлы CSV, XLSX, JSON и HTML. Данные могут читаться из одного файла или из серии файлов. Входные и выходные файлы могут быть в архиве `zip`.

## Основные возможности

Ниже пример спецификаций из тестового конфиг-файла `dfifo_test.py`. В спецификациях
* словарь `fi` задает параметры входного файла, который ищется в директории `IN_DIR`,
* словарь `fo` задает параметры выходного файла, который сохраняется в директории `OUT_DIR`.

Расширение входного и выходного файлов в параметре `"file"` задает формат файла.

```
IN_DIR = os.path.join(os.path.dirname(__file__), '..', 'in')
OUT_DIR = os.path.join(os.path.dirname(__file__), '..', 'out')

specs = {
    "csv_xlsx": {
        "fi": {"file": "test.csv"},
        "fo": {
            "file": "dfifo_test_csv.xlsx",
            "header": ["Name", "А2", "А3", "Code"]
        }
    },
    "csv_parts_json": {
        "fi": {"file": "test_000???.csv"},
        "fo": {
            "file": "dfifo_test_csv_parts.json",
            "header": ["Name", "А2", "А3", "Code"]
        }
    },
    "csv_split_json": {
        "fi": {"file": "test.csv"},
        "fo": {
            "file": "dfifo_test_csv_split_%(seqn)06i.json",
            "header": ["Name", "А2", "А3", "Code"],
            "rows_per_file": 100
        }
    },
    "csv_selected_csv": {
        "fi": {
            "file": "test.csv",
            "transformer": \
                lambda row: (row[3], row[0]) if row[0][0] in 'AEIOU' else None
        },
        "fo": {
            "file": "dfifo_test_csv_selected.csv",
            "header": ["code", "name"]
        },
    }
    ...
}
```

Параметр `"fo/header"` задает
* заголовки столбцов для выходного файла XLSX,
* имена полей объекта JSON для выходного файла JSON,
* имена полей для первой строки выходного файла CSV.

Параметр `"file"` спецификации `"csv_parts_json"` задает [glob-паттерн](https://docs.python.org/3/library/glob.html) для серии загружаемых файлов. Соответствующие файлы загружаются в алфавитном порядке.

Параметр `"fo/file"` спецификации `"csv_split_json"` задает [шаблон в стиле `printf`](https://docs.python.org/3/library/stdtypes.html#printf-style-string-formatting), на основании которого во время выполнения формируется имя выходного файла. Для подстановки контекстных значений в шаблоне можно использовать следующие имена (в скобках):
* `date` – текущая дата в ISO формате `'%Y-%m-%d`,
* `datetime` – текущие дата и время в формате `'%Y-%m-%d-%H-%M-%S'`:
* `seqn` – номер файла в выгружаемой серии файлов ограниченного размера,
* `user` – имя пользователя, заданного в командной строке с помощью ключа `--user`, или имя пользователя ОС, выполняющего утилиту.

Параметр `"rows_per_file"` задает количество строк, которое записывается в один файл. Так как входной файл `test.csv` содержит более 200 строк, то в результате выполнения спецификации `"csv_split_json"` будет создано 3 файла с именами, сформированными по шаблону из параметра `"file"`.

Параметр `"fi/transformer"` спецификации `"csv_selected_csv"` определяет функцию для преобразования кортежа значений, представляющих строку CSV, в кортеж значений для передачи в выходной файл.

Передача данных в выходной файл из входных файлов форматов XLSX, CSV и других текстовых файлов выполняется построчно; из файла формата JSON – пообъектно в порядке расположения объектов в массиве JSON.

Обработка строк утилитой `dfifo` опционально предусматривает

1. проверку, удовлетворяет ли каждая очередная строка некоторому условию; по результату проверки строка либо передается в выходной файл либо игнорируется;
2. формирование списка значений полей для передачи в выходной файл – в случае загрузки из файла JSON или в случае, когда не нужно загружать все поля строки.

При этом простейшая спецификация для `dfifo` 
* содержит только имена входного и выходного файлов,
* предписывает передать все строки из входного в выходной файл.

Познакомьтесь с тем, как используются параметры в тестовом конфиг-файле `dfifo_test.py`.

## Отбор строк и полей

Опциональный параметр спецификации `"skip_lines"` позволяет задать количество строк, которые нужно пропустить в начале загружаемого файла. Например, значение `1` позволяет пропустить первую (вероятно, заголовочную) строку при чтении файлов CSV или XLSX.

Опциональный параметр спецификации `"transformer"` задает функцию, которая позволяет
* загружать только указанные вами поля,
* загружать только строки, соответствующие заданному вами условию.

Параметр `"transformer"` может задавать лямбда-функцию или обычную функцию Python, определенную в конфиг-файле.

Следующая спецификация предписывает избирательную передачу строк и полей в выходной файл:

```
"csv_selected_csv": {
    "tags": ["csv", "selected"],
    "fi": {
        "file": "test.csv",
        "transformer": lambda row: (row[3], row[0]) if row[0][0] in 'AEIOU' else None
    },
    "fo": {
        "file": "dfifo_test_csv_selected.csv",
        "header": ["code", "name"]
    },
}
```

Функция, заданная параметром `"transformer"`, в общем случае
* вызывается для каждой строки загружаемого файла,
* получает в качестве параметра список (list) или кортеж (tuple) всех полей строки, или текстовую строку как есть, если параметр `fi/text_lines` имеет значение True,
* должна возвращать список (list) или кортеж (tuple) полей строки, который пойдет в выходной файл, или текстовую строку, если параметр `fo/text_lines` имеет значение True,
* или возвращать `None`, если данную строку не нужно передавать в выходной файл.

Лямбда-функция в приведенной спецификации возвращает кортеж `(<цифровой код страны>, <название страны>)` для строк, где название страны начинается с гласной буквы, и возвращает `None` в остальных случаях. Таким образом, в выходной файл передается только часть строк и полей из загрузочного файла.

## Обработка данных из файла JSON

Параметр `"transformer"` обязателен в спецификации для загрузки строк из файла JSON. Это связано с тем, что поля объектов JSON могут располагаться в произвольном порядке.

Пример спецификации для трансформации данных из файла формата JSON в файл формата CSV:

```
"json_csv": {
    "tags": ["json"],
    "fi": {
        "file": "test.json",
        "transformer": lambda row: (row["code"], row.get("alpha2"), row.get("alpha3"), row["name"])
    },
    "fo": {"file": "dfifo_test_json.csv"}
}
```

## Распаковка вложенного списка

Каждая строка тестового файла `test_nested_00.csv` содержит два поля:
1. номер этажа торгового центра,
2. список магазинов на этаже.

Вот его содержание:

```
1;ABC,Bonus,Cosmos,Domus,Eidos
2;Focus,Iris
3;Lotus
4;
```

Второе поле, по сути, представляет собой вложенную структуру данных. Этот список необходимо развернуть, или распаковать, так, чтобы передать в выходной файл строки с полями, не содержащими вложенных структур. Для этого в спецификации `"csv_nested_00_json"` параметр `"transformer"` задает функцию, которая для одного кортежа на входе, представляющего строку из файла, возвращает список (!) кортежей, представляющих строки для выходного файла:

```
"csv_nested_00_json": {
    "tags": ['csv', 'unnest'],
    "fi": {
        "file": "test_nested_00.csv",
        "transformer": \
            lambda row: [(row[0], n) for n in row[1].split(',')] if row[1] else [],
    },
    "fo": {
        "file": "dfifo_test_unnested_00.json",
        "header": ["level", "name"],
    }
}
```

Результат выполнения этой спецификации — файл JSON следующего содержания:

```
[
{"level": "1", "name": "ABC"},
{"level": "1", "name": "Bonus"},
{"level": "1", "name": "Cosmos"},
{"level": "1", "name": "Domus"},
{"level": "1", "name": "Eidos"},
{"level": "2", "name": "Focus"},
{"level": "2", "name": "Iris"},
{"level": "3", "name": "Lotus"}
]
```

## Обработка специальных текстовых файлов

Иногда возникает необходимость преобразовать данные из текстовых файлов специальных форматов в один из распространенных форматов, например,
* файл, первые строки которого описывают данные в последующих строках, или
* файл со строками с полями фиксированной длины (без разделителей).

Параметр спецификации `"fi/text_lines": True` предписывает утилите `dfifo` передавать в функцию `"transformer"` строки текстового файла как есть, без предварительного разбиения на поля и преобразования в кортеж.

В этом случае функция, заданная параметром `"transformer"`, выполняет разбор полученной текстовой строки и возвращает список (list) или кортеж (tuple) значений полей для передачи в выходной файл.

Тестовый файл `test_xxx.dat` содержит уже знакомый нам справочник стран мира в строках следующего формата:
1. двухбуквенный код страны – позиции с 1 по 2,
2. трехбуквенный код страны – позиции с 3 по 5,
3. цифровой код страны – позиции с 6 по 8,
4. название страны – позиции с 9 до конца строки.

Вот пять первых и последних строк этого файла:

```
AFAFG004Afghanistan
AXALA248Aland Islands
ALALB008Albania
DZDZA012Algeria
ASASM016American Samoa
...
WFWLF876Wallis and Futuna Islands
EHESH732Western Sahara
YEYEM887Yemen
ZMZMB894Zambia
ZWZWE716Zimbabwe
```

Следующая спецификация передает в выходной файл CSV четыре отдельных поля, извлекая их значения из строк файла с помощью лямбда-функции, заданной параметром `"fi/transformer"`:

```
"dat_xxx_csv": {
    "tags": ["xxx", "dat"],
    "fi": {
        "file": "test_xxx.dat",
        "text_lines": True,
        "transformer": lambda line: (line[0:2], line[2:5], line[5:8], line[8:].strip())
    },
    "fo": {
        "file": "dfifo_test_dat_xxx.csv"
    }
},
```

## Три вида функций `transformer`

Функции, задаваемые параметром спецификации `"fi/transformer"`, могут быть трех видов:

| Вид функции                | Описание                                                                                                                                                                                       |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<func>(row)`              | `row` - список/кортеж значений полей или строка текстового файла как есть, если `"fi/text_lines": True`                                                                                        |
| `<func>(line_no, row)`     | `line_no` - номер строки файла, `row` - список/кортеж значений полей или строка текстового файла как есть, если `"fi/text_lines": True`                                                        |
| `<func>(run, row_no, row)` | `run_no` - уникальный идентификатор обработки, `row_no` - номер обрабатываемой строки, `row` - список/кортеж значений полей или строка текстового файла как есть, если `"fi/text_lines": True` |

При чтении данных из единственного входного файла номер строки файла `line_no` и номер обрабатываемой строки `row_no` совпадают.

Если на входе серия файлов, номер строки файла `line_no` возобновляет счет с `1` для каждого файла в серии, а номер обрабатываемой строки `row_no` непрерывно возрастает на протяжении чтения всех файлов серии.

Функции вида  `<func>(line_no, row)` позволяют обрабатывать специальные текстовые файлы, которые в первых строках содержат метаданные, необходимые для правильной обработки последующих строк. См. пример обработки файла `test_xxx.csv` с помощью спецификации `"xxx_csv"` в тестовом конфиг-файле.

Функции вида `<func>(run_no, row_no, row)` позволяют обрабатывать серию входных файлов.

## Аргументы командной строки

```
$ ./dfifo.py --help
usage: dfifo.py [-h] [-v] [-d] [-f] [-i FI] [-o FO] [-u USER] cfg_file [spec]

Save data from input file(s) to output file(s), as specified in cfg-file specs.

positional arguments:
  cfg_file              cfg-file name
  spec                  spec name, defaults to "all"

options:
  -h, --help            show this help message and exit
  -v, --version         show program's version number and exit
  -d, --delete          delete processed input file(s)
  -f, --force           process input file(s) unconditionally
  -i FI, --fi FI        input file(s)
  -o FO, --fo FO        output file(s)
  -u USER, --user USER  set username

Thanks for using dfifo.py!
```

Обязательно только имя конфиг-файла `cfg_file`.

Если задано имя спецификации `spec`, то выполняется указанная спецификация. Иначе – все спецификации из конфиг-файла.

Ключ `-d` или `--delete` предписывает утилите `dfifo` удалить входной файл после того, как он успешно обработан. Файл не будет удален, если в ходе обработки возникнет ошибка.

Ключ `-f` или `--force` предписывает утилите `dfifo` читать входной файл даже в том случае, если обработка этого файла уже была выполнена ранее. Утилита `dfifo` запоминает время обновления обработанных файлов в служебном файле `~/.<cfg-file>.json` и по умолчанию, то есть, без ключа `-f`, обрабатывает только не обработанные ранее файлы.

Ключ `-i` или `--fi` позволяет задать входной файл вместо файла, указанного в параметре спецификации `fi/file`. Расширение файла определяет формат файла.

Ключ `-o` или `--fo` позволяет задать выходной файл вместо файла, указанного в параметре спецификации `fо/file`. Расширение файла определяет формат файла.

Ключ `-u` или `--user` позволяет передать имя пользователя, которое можно использовать в имени выходного файла.

## Параметры конфиг-файла

Параметры конфиг-файла суть переменные с именами в верхнем регистре, задающие контекст для выполнения спецификаций из данного конфиг-файла. См. также [Структура конфиг-файлов](config.ru.md).

Параметры конфиг-файла для `dfifo` приведены ниже.

| Параметр           | Значение по умолчанию                    | Описание                                                    |
| ------------------ | ---------------------------------------- | ----------------------------------------------------------- |
| `DEBUGGING`        | `False`                                  | Режим отладки?                                              |
| `LOGGING`          | = DEBUGGING                              | Писать в лог-файл?                                          |
| `PARALLEL_WORKERS` | 1                                        | Количество потоков для выполнения спецификаций.             |
| `LOG_DIR`          | `./`                                     | Директория для лог-файлов.                                  |
| `IN_DIR`           | `./`                                     | Директория для входных файлов.                              |
| `OUT_DIR`          | `./`                                     | Директория для выходных файлов.                             |
| `ENCODING`*        | `locale.getpreferredencoding()`          | Кодировка загружаемых файлов.                               |
| `CSV_DIALECT`*     | `excel`                                  | Диалект CSV, определенный модуле Python `csv`, или `naive`. |
| `CSV_DELIMITER`*   | `csv.get_dialect(CSV_DIALECT).delimiter` | Разделитель полей CSV.                                      |
\* параметр конфиг-файла, помеченный звездочкой, на уровне спецификации может быть переопределен соответствующим параметром спецификации.

Помимо диалектов CSV, определенных в модуле Python `csv`, параметр `CSV_DIALECT` позволяет задать диалект `"naive"`. Этот диалект предполагает, что в CSV файле значения полей записаны как есть: без экранирования спецсимволов и без заключения в кавычки. При этом отсутствие в значениях полей символов-разделителей это ответственность тех, кто использует такие файлы.

## Параметры спецификации

Спецификации находятся в конфиг-файле в словаре (dict) `specs` и содержат **параметры спецификации**. См. также [Структура конфиг-файлов](config.ru.md).

Параметры спецификации для `dfifo` приведены ниже. Если специально не оговорено, то параметр спецификации является необязательным и может быть опущен.

| Параметр спецификации     | Описание                                                                                                                                                                                                                                                        |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `"tags"`                  | Список (list) тегов для выбора данной спецификации в командной строке.                                                                                                                                                                                          |
| `"doc"`                   | Краткое описание/комментарий к спецификации.                                                                                                                                                                                                                    |
| `"fi/force"`              | Обрабатывать данные из входного файла безусловно.                                                                                                                                                                                                               |
| `"fi/file"`               | Имя входного файла, расширение задает формат данных. Имя файла может быть [glob-паттерном](https://docs.python.org/3/library/glob.html) для серии входных файлов. Может быть переопределено ключом командной строки `-fi`.                                      |
| `"fi/skip_lines"`         | Количество (`int`) строк, которые необходимо пропустить в начале загружаемого файла.                                                                                                                                                                            |
| `"fi/skip_bad_files"`     | Продолжать обработку серии файлов, если встретился файл с ошибками.                                                                                                                                                                                             |
| `"fi/encoding"`           | Кодировка загружаемого файла. По умолчанию определяется параметром конфиг-файла `ENCODING`.                                                                                                                                                                     |
| `"fi/csv_dialect"`        | Диалект формата CSV. По умолчанию определяется параметром конфиг-файла `CSV_DIALECT`.                                                                                                                                                                           |
| `"fi/csv_delimiter"`      | Разделитель полей формата CSV. По умолчанию определяется параметром конфиг-файла `CSV_DELIMITER`.                                                                                                                                                               |
| `"fi/text_lines"`         | Читать строки текстового файла как есть, невзирая на расширение файла.                                                                                                                                                                                          |
| `"fi/transformer"`        | Функция для преобразования строки входного файла в строку выходного.                                                                                                                                                                                            |
| `"fo/file"`               | Имя выходного файла, расширение задает формат данных. Используйте [шаблон в стиле `printf`](https://docs.python.org/3/library/stdtypes.html#printf-style-string-formatting) для серии выходных файлов. Может быть переопределено ключом командной строки `-fo`. |
| `"fo/rows_per_file"`      | Количество (`int`) строк, записываемых в один файл – для выгрузки данных в серию файлов небольшого размера; -1 означает создание выходного файла для каждого входного.                                                                                          |
| `"fo/header"`             | Список (list) имен полей выходного файла.                                                                                                                                                                                                                       |
| `"fo/text_lines"`         | Писать текстовые строки как есть, невзирая на расширение текстового файла.                                                                                                                                                                                      |
| `"fo/csv.encoding"`       | Кодировка файла формата CSV. По умолчанию определяется параметром конфиг-файла `ENCODING`.                                                                                                                                                                      |
| `"fo/csv.dialect"`        | Диалект формата CSV. По умолчанию определяется параметром конфиг-файла `CSV_DIALECT`.                                                                                                                                                                           |
| `"fo/csv.delimiter"`      | Разделитель полей формата CSV. По умолчанию определяется параметром конфиг-файла `CSV_DELIMITER`.                                                                                                                                                               |
| `"fo/csv.dec_separator"`  | Десятичный разделитель для чисел в файле формата CSV. По умолчанию точка.                                                                                                                                                                                       |
| `"fo/json.encoding"`      | Кодировка файла формата JSON. По умолчанию определяется параметром конфиг-файла `ENCODING`.                                                                                                                                                                     |
| `"fo/json.template"`      | Файл шаблона Jinja2 для получения файла формата JSON. По умолчанию используется встроенный шаблон, аналогичный `cfg/dfifo_sample.json.jinja`.                                                                                                                   |
| `"fo/html.encoding"`      | Кодировка файла формата HTML. По умолчанию определяется параметром конфиг-файла `ENCODING`.                                                                                                                                                                     |
| `"fo/html.template"`      | Файл шаблона Jinja2 для получения файла формата HTML. По умолчанию используется встроенный шаблон, аналогичный `cfg/dfifo_sample.html.jinja`.                                                                                                                   |
| `"fo/html.title"`         | Заголовок для файла формата HTML. По умолчанию имя спецификации.                                                                                                                                                                                                |
| `"fo/html.dec_separator"` | Десятичный разделитель для чисел в файле формата HTML. По умолчанию точка.                                                                                                                                                                                      |
