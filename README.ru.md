# dbang

	версия 0.4.0

Читать на других языках: [английский](README.md).

Утилиты **dbang**
*  проверяют логическую полноту и целостность данных в БД
	* [`ddiff.py`](doc/ddiff.ru.md) - выполняет запросы к двум БД и формирует отчет о расхождениях,
	* [`dtest.py`](doc/dtest.ru.md) - выполняет запросы к БД и формирует отчет о найденных проблемах,
* выгружают и загружают данные из/в БД согласно спецификациям в конфиг-файле
	* [`dget.py`](doc/dget.ru.md) - выгружает данные из БД в файлы CSV, XLSX, JSON, HTML,
	* [`dput.py`](doc/dput.ru.md) - загружает данные из файлов CSV, XLSX, JSON в таблицы БД,
* формируют и отправляют е-мейл согласно спецификациям в конфиг-файле
	* [`hedwig.py`](doc/hedwig.ru.md) - отправляет е-мейл на основе файлов (например, созданных `ddiff` или `dget`),
* преобразуют данные из файлов в файлы CSV, XLSX, JSON, HTML, согласно спецификациям в конфиг-файле
    * [`dfifo.py`](doc/dfifo.ru.md) - читает из, фильтрует, преобразует и пишет данные в файлы.

Утилиты работают с СУБД PostgreSQL, Oracle, MySQL, MS SQL Server и SQLite посредством DBAPI модулей Python.

## Установка

Утилиты **dbang** написаны на Python 3 и зависят от нескольких пакетов Python, указанных в файле [`requirements.txt`](requirements.txt).

Для установки утилит **dbang**
1. Скачайте [сжатый архив последнего релиза](https://github.com/andorei/dbang/releases/latest).
2. Распакуйте его в директории по вашему выбору.
3. Установите с помощью `pip` необходимые зависимости из файла [`requirements.txt`](requirements.txt)
```
cd dbang-<version>
pip install -r requirements.txt
```

## Использование

Утилиты запускаются командной строкой

```
python <utility>.py [options] <config-file> [<spec> | all]
```

где 
* `<utility>` – одно из `ddiff`, `dtest`, `dget`, `dput`, `hedwig` или `dfifo`,
* `options` – опции командной строки, свои для каждой утилиты,
* `<config-file>` – имя конфиг-файла,
* `<spec>` - имя спецификации или тэга спецификации из конфиг-файла. Если `<spec>` не указано, то выполняются все спецификации из конфиг-файла.

Чтобы познакомиться со всеми опциями утилиты, запустите ее с опцией `--help`.

Разумеется, на Linux вы можете сделать утилиты исполняемыми и запускать их просто по имени, не вызывая python в командной строке.

Утилиты **dbang**
* выполняют спецификации из конфиг-файлов, написанных на Python,
* пишут информационные и отладочные сообщения в лог-файлы, если в конфиг-файле включен режим логирования или отладки,
* сохраняют результаты работы и логи в директориях, заданных в конфиг-файле переменными `OUT_DIR` и `LOG_DIR`, соответственно, или в текущей директории, если эти переменные не определены.

Выполнение спецификаций из конфиг-файла можно распараллелить, запуская их в в разных потоках (threads). Конфигурируемое кол-во потоков для `ddiff`, `dget`, `dput`, `dtest` ограничено максимум 8 потоками, так как каждый поток открывает отдельное соединение с БД. Количество потоков для `dfifo` ограничено максимум 64 потоками.

## Примеры использования

Директория `test` содержит тестовые конфиг-файлы, написанные с целью тестирования и с целью демонстрации возможностей утилит. Все тестовые конфиг-файлы используют источники данных, определенные в конфигурационном файле `sources.py`, который отсутствует сразу после установки.

Создайте файл `sources.py` в директории `test` самостоятельно, скопировав имеющийся файл `sample_sources.py`. После этого вы сможете запустить утилиты с конфиг-файлами, которые работают с локальной БД sqlite3:

```
cd dbang-<version>
./ddiff.py test/ddiff_test_sqlite
./dtest.py test/dtest_test_sqlite
./dget.py test/dget_test_sqlite
./dput.py test/dput_test_sqlite all
./dfifo.py test/dfifo_test all
```

В директории `out` вы найдете
* отчет о расхождениях в данных, сформированный `ddiff`,
* отчет о качестве данных, сформированный `dtest`,
* файлы с данными, извлеченными `dget` из базы данных.

В директории `log` вы найдете лог-файлы каждой из утилит.

Для выполнения тестовых конфиг-файлов для СУБД PostgreSQL, Oracle, MySQL или MS SQL Server, отредактируйте в файле `sources.py` параметры подключения для источника `"postgresql_source"`, `"oracle_source"`, `"mysql_source"` или `"mssql_source"`.  После этого утилиты, запущенные с соответствующими конфиг-файлами, подключатся к указанной вами БД.

Прежде чем запустить `hedwig.py` с тестовым конфиг-файлом `hedwig_test.py`, присвойте правильные значения переменным `MAIL_SERVER` и `MAIL_TO` в конфиг-файле (или в файле `sources.py`, который импортируется в конфиг-файл). После этого выполните команду

```
./hedwig.py test/hedwig_test
```

и проверьте входящую почту.

## Что дальше

Чтобы эффективно пользоваться утилитами **dbang**, вам нужно научиться создавать конфиг-файлы со спецификациями.

В качестве шаблона для вашего конфиг-файла можно взять один из файлов-заготовок в директории `cfg` или один из тестовых конфиг-файлов в директории `test`. Скопируйте его под новым именем в директорию `cfg` и внесите необходимые изменения.

Вы можете настроить запуск утилит по расписанию (с помощью `crontab` в Linux или Планировщика заданий в Windows) с последующей рассылкой сформированных отчетов и выгруженных файлов по электронной почте утилитой `hedwig`.
